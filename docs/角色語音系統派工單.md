# Auris 角色語音系統派工單

> 主計畫：`docs/角色語音系統開發計畫.md`
> 日期：2026-07-23
> 用途：交給實作模型逐批施工，不得一次完成全部批次

## 派工順序

| 順序 | 交付內容 | 開始條件 |
|---|---|---|
| A | 安全基礎、provider registry、ElevenLabs adapter、測試 | 主計畫核准 |
| B | 全域語音服務設定與聲音清單 | A 的資安測試全部通過 |
| C | 每角色 Voice ID、試聽與匯入／匯出隔離 | B 通過手機版操作驗收 |
| D | 聊天訊息播放、Blob 快取與清除連動 | C 的資料相容測試通過 |

上一批未通過前，不得開始下一批。

## 批次 A 任務指令

可把以下整段直接交給實作模型：

> 請完整閱讀根目錄 `AGENTS.md`、`docs/角色語音系統開發計畫.md` 與 `docs/角色語音系統派工單.md`。
>
> 只施工主計畫的「批次 A：安全基礎與服務商轉接層」，不要提前製作 UI、角色設定、聊天室播放、音訊快取或自動語音。
>
> 動工前先檢查目前分支與工作區，保留所有既有修改及未追蹤檔案。
>
> 主計畫第 6 節所有資安防線均為強制需求：
> - API Key、端點、帳戶識別、provider、voice ID、供應商模型與驗證狀態，不得出現在角色匯出、全域備份、診斷、錯誤、console、URL 或測試快照。
> - 匯出必須採 allowlist 建立新物件；匯入也必須採 allowlist，在寫入前丟棄所有語音 API 與角色聲音綁定欄位。
> - 首期只允許硬編碼的 ElevenLabs 官方 HTTPS origin，不提供 Custom Base URL。
> - Voice ID 必須限制長度及字元，拒絕 URL、斜線與控制字元。
> - 所有請求必須有 timeout 與 AbortController。
> - 錯誤訊息必須淨化，不得直接保存完整 response body 或 headers。
> - 音訊回應必須限制 MIME、基本檔案簽章及最大大小。
> - 401、403、429、額度不足與網路錯誤必須分開處理。
> - 計費狀態不明時不可自動重試。
>
> 架構需提供 provider registry 與 ElevenLabs adapter，View 不得直接組裝 URL 或 Authorization header。未知 provider 必須在發出網路請求前拒絕。
>
> 完成後執行新增單元測試、既有相關測試與 `/verify-app`。準備 commit 前才執行 `/bump`；未經使用者要求不要 commit、push 或開始批次 B。
>
> 最後依本派工單的回報格式提供修改檔案、測試證據、資安證據、殘留風險及下一批前置條件。

## 批次 A 必交測試

1. 未註冊 provider 在任何 fetch 前被拒絕。
2. ElevenLabs URL、method、headers、model 與語音參數正確。
3. Key 缺失時不送出請求。
4. Voice ID 空值、過長、URL、斜線與控制字元均被拒絕。
5. timeout 會 abort；元件或呼叫端取消也會 abort。
6. 401、403、429、quota、timeout、離線錯誤轉為安全且可理解的錯誤碼。
7. 錯誤物件、console mock 與診斷 ring buffer 中不存在測試 Key。
8. 全域備份、角色匯出與診斷匯出中不存在 TTS Key、端點、provider、voice ID、供應商模型或帳戶識別。
9. 不允許角色匯入或全系統還原資料寫入、覆蓋任何語音 API、角色聲音綁定或服務商 origin。
10. 錯誤 MIME、偽造音訊簽章及超大回應被拒絕。
11. 同一 AbortSignal 可正確取消列表與合成請求。
12. 不因網路錯誤以外的錯誤自動重試。

測試 Key 必須使用明顯假值，例如 `xi-demo-not-a-real-key`，不得使用任何真實帳號資料。

## 每批回報格式

實作模型必須回報：

1. 本批實際完成項目。
2. 本批刻意未做項目。
3. 修改檔案清單及資料結構變化。
4. 測試指令、通過數量及失敗項目。
5. API Key 與角色聲音 API 綁定不進入角色匯出、全域備份、診斷、錯誤及日誌的實際證據。
6. 網路端點 allowlist、timeout 與輸入驗證證據。
7. 手機版或瀏覽器實測證據；批次 A 若無 UI，說明不適用。
8. 尚未消除的資安與相容性風險。
9. 下一批開始前需要使用者確認的事項。

只寫「已完成」或「測試通過」不算驗收證據。

## 不得自行改變的產品決策

- Auris 維持純前端 BYOK。
- API 費用由使用者的服務商帳號支付。
- 首期只支援 ElevenLabs。
- 每個角色保存 `provider + voiceId`，Voice ID 不跨服務商通用。
- 上述角色聲音綁定只保存在目前裝置；角色匯出、全系統備份均不攜帶，角色匯入與全系統還原也不得接受或覆蓋這些欄位。
- 首期只在使用者主動試聽或播放時產生語音。
- 文字是主要資料，音訊是可清除、可重建的衍生資料。
- 首期不做聲音複製、授權樣本、Voice Design、自動語音及自訂代理。

## 必須明確回報的殘留風險

Auris 是靜態 PWA；長期 API Key 存在瀏覽器時，無法獲得後端秘密儲存的安全性。批次 A 必須：

- 在資料模型與 UI 契約中保留風險告知。
- 建議使用只有所需權限的獨立 Key。
- 建議在 ElevenLabs 設定信用額度或用量上限。
- 提供日後清除 Key 的服務層能力。
- 不得宣稱本機儲存等於完全保密。

如果實測發現 ElevenLabs CORS、服務條款或端點不允許所需的瀏覽器呼叫，立即停止並回報；不得暗中加入第三方代理或把 Key 傳到非官方網域。
