## 文件維護規則

1. 已上線能力只寫進 `product_feature_list.md`。
2. 未完成項目只寫進 `ROADMAP.md`；完成後移出 Roadmap，版本成果記進進度總覽。
3. 技術現況只寫進 `ARCHITECTURE.md`；版本時間線以進度總覽為準。
4. 審查報告必須在檔頭維護「目前狀態表」，原始證據保留但不可冒充最新狀態。
5. 密碼、API key、service account、個人絕對路徑不得出現在任何 tracked 文件。
