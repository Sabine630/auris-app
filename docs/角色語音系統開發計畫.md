# Auris 角色語音系統開發計畫

> 狀態：規劃稿，尚未施工
> 日期：2026-07-23
> 首期供應商：ElevenLabs
> 付費模式：BYOK（使用者自備 API Key、由使用者帳號支付）

## 1. 目標

建立可擴充的角色語音基礎，使每個角色能綁定不同服務商的聲音，並依序支援：

1. 在全域設定中連接語音服務商。
2. 載入該帳號可用聲音並試聽。
3. 每個角色選擇自己的聲音。
4. 用角色聲音播放既有聊天訊息。
5. 後續讓角色主動傳語音訊息。
6. 後續朗讀日記、夢境及音訊貼文。

首期不做聲音複製、授權樣本上傳、App 內 Voice Design、影片／音樂貼文。

## 2. 已確定的產品決策

- Voice ID 不是跨平台通用值，必須與 `provider` 一起保存。
- 全域設定保存各服務商的連線設定；角色只引用服務商與 Voice ID。
- 首期只實作 ElevenLabs，但資料結構與介面不得寫死單一供應商。
- API 費用由使用者自己的服務商帳號支付，Auris 不代墊、不代收。
- 首期只在使用者主動按下播放或試聽時呼叫 TTS，避免意外扣費。
- 每次產生語音都必須保留文字內容；音訊失敗不能破壞原訊息。
- 沒有設定 TTS 時，原有文字聊天完全不受影響。
- 不提供任意 Custom Base URL，避免金鑰被送到偽造端點。

## 3. 使用流程

### 3.1 連接服務商

`設定 → 角色語音 → ElevenLabs`

1. 顯示用途、費用歸屬及資料傳送告知。
2. 使用者輸入 ElevenLabs API Key。
3. Auris 呼叫「列出聲音」端點驗證連線。
4. 成功後只顯示「已連接」及遮罩後 Key，不回顯完整內容。
5. 顯示帳號可用聲音，允許逐一試聽。

### 3.2 為角色選擇聲音

`編輯角色 → 進階設定 → 角色聲音`

1. 開啟「啟用角色聲音」。
2. 選擇服務商；首期只有 ElevenLabs。
3. 從已載入的聲音清單選擇，或手動輸入該服務商的 Voice ID。
4. 設定語速；穩定度、相似度收進「進階調整」。
5. 使用固定短句試聽。
6. 試聽成功後才標示「已驗證」，使用者仍可明確選擇儲存。

### 3.3 播放既有訊息

1. 長按角色訊息，選擇「用角色聲音播放」。
2. 第一次播放呼叫 TTS；同一文字、聲音及參數優先使用本機快取。
3. 播放列提供播放／暫停／停止狀態。
4. 文字仍是訊息的主資料，音訊只是可重建的衍生快取。

## 4. 資料模型

### 4.1 全域服務商設定

API Key 不放入角色資料。整個 `ttsProviders` 只屬於目前裝置的本機設定，
不得進入角色匯出、全系統備份或診斷匯出。

```js
ttsProviders: {
  elevenlabs: {
    enabled: true,
    apiKey: 'secret-local-only',
    model: 'eleven_flash_v2_5',
    connectedAt: 0,
    lastValidatedAt: 0
  }
}
```

### 4.2 角色聲音設定

```js
voiceProfile: {
  enabled: true,
  provider: 'elevenlabs',
  voiceId: 'JBFqnCBsd6RMkjVDRZzb',
  voiceName: 'George',
  model: '',
  speed: 1,
  settings: {
    stability: 0.5,
    similarity: 0.75
  },
  verifiedAt: 0
}
```

規則：

- `voiceName` 只供顯示，以 `voiceId` 為實際呼叫依據。
- `model` 留空時繼承服務商全域模型。
- `voiceProfile` 是目前裝置的本機綁定資料，**整個欄位不得進入角色匯出或全系統備份**。
- 角色匯入與全系統還原也必須主動丟棄 `voiceProfile`、`ttsProviders`、`provider`、`voiceId`、供應商模型與驗證狀態；不可因匯入檔帶有欄位便寫回本機。
- 目前預設所有角色聲音相關欄位都不匯出。未來若要讓「不含供應商資訊的聲音描述」跟著角色移轉，必須另開資料格式與資安審查，不得直接沿用 `voiceProfile`。
- 未支援的 `provider` 必須顯示「此裝置尚未支援」，不可自動改用其他服務商。

### 4.3 音訊快取索引

```js
{
  id: 'tts_<hash>',
  provider: 'elevenlabs',
  voiceId: '...',
  model: '...',
  textHash: '...',
  settingsHash: '...',
  mime: 'audio/mpeg',
  bytes: 0,
  createdAt: 0,
  lastUsedAt: 0,
  blob: Blob
}
```

快取 Key 必須包含服務商、Voice ID、模型、文字及會影響聲音的參數。音訊存 Blob，不使用 base64。

## 5. 程式架構

建議新增：

```text
auris-vue/src/services/tts/
├── index.js
├── providerRegistry.js
├── audioCache.js
└── providers/
    └── elevenlabs.js
```

供應商共同介面：

```js
{
  id,
  validateConfig(config),
  listVoices(config, { signal }),
  synthesize({ config, voiceId, model, text, settings, signal }),
  normalizeError(error)
}
```

禁止 View 直接組裝供應商 URL 或 Authorization header。所有網路呼叫集中在 provider adapter，日後新增 MiniMax 時不得修改聊天室核心流程。

既有 `services/speech.js`：

- 保留 `stripActionText()` 作為朗讀前文字清理。
- 瀏覽器 `speechSynthesis` 不作為高品質角色聲音的 fallback。
- 可以保留舊測試，但新 TTS 狀態與播放控制應由新 service 負責。

可能新增／修改的畫面：

- `SettingsView.vue`：新增「角色語音」入口。
- 新增 `VoiceSettingsView.vue`：服務商、Key、連線、聲音清單與預設試聽。
- `CharEditView.vue`：進階設定加入角色聲音區。
- `ChatRoomView.vue`：長按選單及播放狀態。
- `router/index.js`、`App.vue`：新增隱藏底部導覽的設定子頁。

## 6. 資安威脅與強制防線

### 6.1 長期 API Key 無法在純前端完全保密

Auris 是靜態 PWA。只要 Key 儲存在瀏覽器，使用者本人、瀏覽器擴充套件、惡意依賴或 XSS 都可能讀取；前端程式無法把長期密鑰變成真正的秘密。ElevenLabs 官方亦不建議在瀏覽器暴露 API Key。

首期若維持「資料不經 Auris 中間伺服器」的 BYOK 架構，必須：

- 在輸入 Key 前清楚揭露風險。
- 建議使用者建立僅有 TTS／讀取 Voices 權限的獨立 Key。
- 建議在服務商端設定信用額度或用量上限。
- UI 永不再次顯示完整 Key。
- 提供「清除語音 Key」按鈕。
- 不宣稱 Key 能在純前端被完全保護。

未來若要降低風險，只能另行評估：

- 服務商 OAuth／短效 token。
- 使用者信任的自架 relay。
- Auris 後端代管密鑰；這會改變「資料只存在本機」承諾，不能默認導入。

### 6.2 Key 外洩防線

- API Key、服務商設定、端點、帳戶／專案識別、`provider`、`voiceId`、供應商模型、驗證狀態及音訊快取索引，不得進入角色匯出、全域備份、診斷 JSON、錯誤 ring buffer、console、toast、URL 或 analytics。
- 匯出採明確 allowlist 建立新物件，不可直接序列化 store 後再刪除少數欄位。
- 匯入採 schema allowlist；即使是人為加工的匯入檔，也必須在寫入 IndexedDB 前丟棄所有語音 API 與角色聲音綁定欄位。
- 錯誤內容須白名單化，不直接顯示或記錄完整 response body／request headers。
- 新增測試，掃描所有匯出物與診斷內容不存在 Key、voice ID、provider 綁定或端點。
- 沿用 `check-secret-leak` hook，新增 TTS Key 測試樣本時只能使用明確假值。
- Key 欄位使用 `type="password"`，關閉瀏覽器自動完成建議。

### 6.3 網路與端點防線

- 首期只允許硬編碼的官方 ElevenLabs HTTPS origin。
- 不接受匯入資料覆蓋 API base URL。
- 設定連線及合成都必須有 AbortController timeout。
- 限制 voiceId 長度與字元集；拒絕控制字元、斜線及 URL。
- 不把 Voice ID 拼成未經 `encodeURIComponent` 的路徑。
- 供應商回傳的 voice name 只以 Vue 文字插值顯示，禁止 `v-html`。

### 6.4 音訊回應防線

- 限定允許的 MIME，例如 `audio/mpeg`、`audio/wav`、`audio/ogg`。
- 設定單次最大回應大小；超限立即中止且不落庫。
- 檢查基本檔案簽章，不只相信 `Content-Type`。
- Blob 使用 object URL 播放，結束後 `URL.revokeObjectURL()`。
- 音訊不可執行，不插入 HTML，不接受 SVG／data:text。
- 快取採 LRU 或總量上限，避免 IndexedDB 被無限塞滿。

### 6.5 費用與濫用防線

- 首期禁止自動朗讀所有回覆。
- 試聽及播放按鈕要有 loading lock，避免連點重複扣費。
- 相同請求進行中要去重。
- 每次送出的文字設上限，長文先顯示字數及確認。
- 本機保存當日生成次數與估算字元數，提供用量提醒；它只供提醒，不宣稱等同供應商帳單。
- HTTP 401／403／429／quota error 分開處理，不自動無限重試。
- 網路錯誤最多重試一次；計費狀態不明時不自動重試。

### 6.6 隱私告知

播放角色語音時，訊息文字會傳送給選定的 TTS 服務商。首次啟用必須告知：

- 傳送內容：要朗讀的文字、Voice ID、語音參數。
- 接收者：選定的服務商。
- 費用：由使用者自己的服務商帳號支付。
- Auris 本機保存：連線設定、角色 Voice ID、可清除的音訊快取。

## 7. 分批施工

### 批次 A：安全基礎與服務商轉接層

- 完成資料模型與 provider registry。
- 完成 ElevenLabs adapter。
- 完成 Key 的本機保存、遮罩、清除，以及所有語音 API／角色聲音綁定資料的匯出排除與匯入丟棄。
- 完成 timeout、輸入驗證、錯誤淨化、回應大小與 MIME 防線。
- 單元測試不碰 UI。

驗收：

- 正確 Key 可以列出聲音。
- 無效／超額／網路失敗有可理解且不洩密的錯誤。
- 角色匯出、全域備份及診斷輸出中找不到 Key、provider、voice ID、端點或供應商模型。
- 人為塞入上述欄位的角色匯入與全系統還原檔，還原後也不能污染本機語音設定。
- 未註冊 provider 無法發出請求。

### 批次 B：服務商設定畫面

- Settings 新增「角色語音」入口。
- 完成服務商說明、Key 設定、連線測試、聲音列表與試聽。
- 顯示費用及資料傳送告知。

驗收：

- 320px 寬度無溢位。
- iOS PWA 可輸入、返回及播放。
- 連點試聽只產生一個請求。
- 離開畫面會停止播放並釋放 object URL。

### 批次 C：角色綁定聲音

- CharEdit 進階設定加入啟用開關、服務商、聲音選擇、Voice ID、語速與進階參數。
- 角色匯入／匯出相容舊資料，但所有語音 API 與角色聲音綁定欄位一律不移轉。
- 未設定全域 Key 時提供明確導引。

驗收：

- 不同角色可以選不同 Voice ID。
- 刪除或停用服務商不會破壞角色資料。
- 舊角色沒有 `voiceProfile` 時正常載入。
- 手動 Voice ID 必須試聽成功或明確標示「未驗證」。

### 批次 D：聊天訊息播放與快取

- 長按選單加入「用角色聲音播放」。
- 完成播放／暫停／停止、請求去重及 Blob 快取。
- 離開聊天室停止播放。

驗收：

- 同一訊息第二次播放命中快取，不重複扣費。
- 修改文字、Voice ID 或語音參數後不誤用舊快取。
- TTS 失敗時文字訊息仍完整。
- 清除角色／聊天／全部資料時，相關音訊快取同步處理。

### 後續批次（不屬於首期）

- 角色主動傳真實語音訊息與逐字稿。
- 模擬語音泡泡。
- 日記／夢境按需朗讀。
- 音訊貼文。
- MiniMax adapter。
- Voice Design 精靈。
- 授權樣本與聲音複製。

## 8. 測試清單

至少新增：

- provider registry：未知 provider 拒絕。
- ElevenLabs request：URL、header、模型、參數、timeout。
- Voice ID 驗證：空值、過長、斜線、URL、控制字元。
- 語音機密與綁定：API Key、端點、帳戶識別、provider、voice ID、供應商模型不出現在角色 export、全域 backup、diagnostics、error、console mock。
- 匯入污染：惡意角色卡與備份內塞入上述欄位時，寫入前全部丟棄。
- 音訊驗證：錯誤 MIME、偽造簽章、超大回應。
- 併發：連點只送一次、停止後 abort。
- 快取：命中、失效、LRU、清除連動。
- 資料遷移：無 voiceProfile 舊角色、未知 provider、新舊備份。
- UI：未連線、連線中、成功、401、429、離線、播放中。

依 `/verify-app` 流程跑完整 vitest、build、dev server 實測；iOS PWA 至少驗證設定頁鍵盤、音訊播放解鎖及返回後停止播放。

## 9. 完成定義

- 使用者能安全地設定、驗證及清除 ElevenLabs Key。
- 使用者能載入聲音、試聽並為每個角色保存 Voice ID。
- 使用者能手動播放角色既有訊息，且第二次命中本機快取。
- 沒有 API Key、API 失敗或額度不足時，文字聊天不受影響。
- API Key 與角色聲音 API 綁定不出現在任何角色匯出、全系統備份、診斷、日誌或 URL。
- 所有新增測試、既有測試與 build 通過。
- 執行 `/bump`，同步版本、進度總覽、架構文件及必要功能清單後才可 commit。

## 10. 明確禁止

- 不可把任何真實 API Key 寫進程式、測試或截圖。
- 不可把 Key、provider、voice ID、端點、帳戶識別或供應商模型放進角色匯出或備份。
- 不可由 View 自行呼叫任意 TTS URL。
- 不可把服務商錯誤 body 原樣寫入診斷。
- 不可默認自動生成所有訊息的音訊。
- 不可在首期加入聲音複製或上傳他人錄音。
- 不可在沒有使用者確認的情況下推送 `main`。
